## nodiscard
C++23에서 반환값을 사용하지 않으면 컴파일러 경고를 일으키는 `[[nodiscard]]` 속성을 사용할 수 있다. 
```
auto lambda = [] [[nodiscard]] (){ return 4; };
```
## Capture List and Classes
- `[this]`는 현재 객체 `(*this)`를 참조로 캡처 (C++17에서 암시적)
- [x = x]는 C++14에서 클래스의 멤버 변수를 값으로 캡처
- [&x = x]는 C++14에서 클래스의 멤버 변수를 참조로 캡처
- [=]는 C++20에서 this 포인터를 값으로 캡처하는 것이 비권장됨
```
class A {
    int data = 1;
    void f() {
        int var = 2; // 지역 변수
        auto lambda1 = [=]() {return var;}; // 값으로 복사, 2를 반환
        auto lambda2 = [=]() {int var = 3; return var; }; // 값으로 캡처, 3반환
        auto lambda3 = [this]() {return data;}; // 참조로 복사, 1을 반환
        auto lambda4 = [*this]() {return data;}; // 값으로 복사 (C++17), 1을 반환
        //auto lambda5 = [data]() {return data;}; // 컴파일 에러. 멤버 변수를 직접 캡처할 수 없음
        auto lambda6 = [data = data]() {return data;}; // 1을 반환
    }
};
```

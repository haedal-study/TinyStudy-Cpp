- C++ Philosophy (철학)
	- Zero Overhead Principle (zero-cost abstraction)
		- Do not sacrifice performance except as a last resort
	- Statically Typed Language
		- Enforce safety at compile time whenever possible
		- The type annotation으로 코드 가독성 향상
		- 컴파일러 최적화 및 런타임 효율성 촉진
		- 사용자가 직접 타입 시스템을 정의할 수 있음
	- 프로그래밍 모델
		- 구획화, 실제 문제를 해결하는 경우에만 기능 추가, 완전한 제어 허용
	- 예측 가능한 런타임(제약 조건 하에서)
		- 가비지 컬렉터 없음, 동적 유형 시스템 없음 → 실시간 시스템
	- 낮은 리소스
		- 낮은 메모리 및 에너지 소비량 → 제한된 하드웨어 플랫폼
	- 정적 분석에 적합
	- 최신 C++ 표준은 이식성이 뛰어남

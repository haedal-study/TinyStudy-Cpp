# C++은 왜 어려운가?
- 하나에 많은 언어 기능
- 광범위한 기능
- 메모리 관리 문제
- 저수준 구현 세부 사항
- 오류
<br></br>
# 역호환성
- 위험한 기본값 및 구조를 제거하거나 변경할 수 없음
- 새로운 기능들은 광범위한 사용자들의 경험을 거친 후에야 수정할 수 없는 결함을 찾음
- 개발자들에게 인지적 부담을 줌
<br></br>
# C++ 대체자
### Rust
- C++로서 성능과 추상화 오버헤드 제로에 초점
- 컴파일 유현에서 제약 조건을 적용
- 플랫폼 간 호환성을 촉진
### Zig
- C의 대체로 오픈 소스 프로그래밍 언어
- 컴파일 타임 제네릭, 리플렉션 및 평가, 크로스 컴파일, 수동 메모리 관리를 지원
- C와 상호 운용 가능하도록 만들어짐
- C/C++ 컴파일러 포함
<br></br>
